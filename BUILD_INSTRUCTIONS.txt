OLYMP PLANNER - BUILD INSTRUCTIONS
==================================

This is a fully functional Android planner application with Olympus-themed UI.

REQUIREMENTS:
- Android Studio (latest version recommended)
- JDK 17 or higher
- Android SDK with compileSdk 36
- Minimum Android API level 26 (Android 8.0)

PROJECT SETUP:
1. Open the project in Android Studio
2. Wait for Gradle sync to complete
3. Ensure you have the required SDK components installed
4. Build and run the application

FEATURES IMPLEMENTED:
✓ Week View - Overview of 7 days with tasks
✓ Day View - Detailed daily planning with time segments
✓ Notes - Text notes and checklists with categories
✓ Themes - Day/Night/Lightning themes with customization
✓ Task Management - Create, edit, delete, complete tasks
✓ Categories - Work, Personal, Leisure
✓ Priorities - Low, Medium, High
✓ Reminders - Local notifications with snooze
✓ Filters - By category, deadlines, all
✓ Beautiful UI - Olympus-themed with marble columns and lightning effects
✓ Animations - Smooth transitions and visual effects
✓ Room Database - Local data persistence
✓ Material Design 3 - Modern UI components

ARCHITECTURE:
- Clean Architecture (Data, Domain, UI layers)
- MVVM pattern with ViewModels
- Repository pattern for data access
- Room for local database
- Jetpack Compose for UI
- Navigation Component
- Kotlin Coroutines & Flow
- Version Catalog for dependencies

NAVIGATION:
- Bottom Navigation Bar with 4 tabs:
  * Week (overview)
  * Day (detailed view)
  * Notes (quick notes)
  * Themes (personalization)

THEME MODES:
- Day: Light mode with sky blue gradient
- Night: Dark mode with midnight colors
- Lightning: Dynamic mode with white lightning effects

PERMISSIONS:
- POST_NOTIFICATIONS: For task reminders
- SCHEDULE_EXACT_ALARM: For precise timing
- VIBRATE: For notification feedback

TO BUILD RELEASE APK:
1. Open terminal in project root
2. Run: ./gradlew assembleRelease (or gradlew.bat on Windows)
3. APK will be in: app/build/outputs/apk/release/

TROUBLESHOOTING:
- If Gradle sync fails, ensure you have the correct JDK version
- If build fails, clean project: Build > Clean Project
- Check that Android SDK is properly configured
- Ensure all dependencies are downloaded

NOTES:
- Database is automatically created on first launch
- No backend required - fully offline app
- All data stored locally using Room
- Notifications require permission grant on Android 13+
